Bonne Pratique de Git

Voici quelques bonnes pratiques à suivre lors de l'utilisation de Git :

  • create a new repository on the command line: voici les commandes a saisir . 
          
      echo "# git-test" >> README.md
      git init
      git add README.md
      git commit -m "first commit"
      git branch -M main
      git remote add origin git@github.com:Elolo-GIT/git-test.git
      git push -u origin main
        
    Création d'un repository : 
        Utilisez la commande git init pour créer un nouveau repository local, ou git clone pour cloner un repository existant.
    
  • Création de branches :
        Utilisez la commande git branch pour créer de nouvelles branches pour le développement de fonctionnalités ou la correction de bugs.
    git checkout -b nom-de-la-branche

    git push origin nom-de-la-branche

  • git push : Utilisez la commande git push pour envoyer vos commits locaux vers un repository distant, comme GitHub ou GitLab, afin de partager votre travail avec d'autres développeurs.
    git add nom du fichier

    git commit -m "message de commit (skip ci)" attention ce sont des crochet et non des parenthèse

    git push origin nom-de-la-branche

  • Commit souvent : Faites des commits fréquents avec des
        messages clairs pour faciliter la compréhension de l'historique du
        projet.
    • Cherry-pick : Utilisez la commande git cherry-pick pour appliquer des commits spécifiques d'une branche à une autre, ce qui vous permet de sélectionner des modifications particulières sans fusionner toute la branche.
  • Utilisez des branches : Travaillez sur des branches
        pour développer de nouvelles fonctionnalités ou corriger des bugs sans
        affecter la branche principale.
    • Rebase : Utilisez la commande git rebase pour réorganiser les commits d'une branche, ce qui permet de maintenir un historique linéaire et propre.
  • Faites des pull requests : Utilisez les pull requests
        pour revoir le code avant de le fusionner dans la branche principale, ce
        qui améliore la qualité du code.
  • Gardez votre dépôt propre : Supprimez les branches
        obsolètes et assurez-vous que votre dépôt ne contient pas de fichiers
        inutiles.
  • Communiquez avec votre équipe : Utilisez les
        commentaires dans les commits et les pull requests pour expliquer vos
        changements et collaborer efficacement avec votre équipe.
